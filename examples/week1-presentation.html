AX (AI Transformation)란?

단순히 AI를 써보는 것에 그치지 않고, 업무 과정 전체를 AI와 협업하는 중심으로 바꾸는 것

기존: 챗봇 사용

검색창처럼 사용

전환점

직접 실험하고 확인

AX: 통합 협업

모든 업무 프로세스에
AI를 녹여내기

"AI에게 일을 시키는 것"이 아니라 "AI와 함께 일하는 것"

AI를 동료로 대하기

도구로 쓸 때

"카드뉴스 만들어줘"
일방적 지시 → 뽑기 운

동료로 대할 때

"이 데이터를 보고 같이 기획해보자"
협업 → 일관된 품질

뭐든 AI로 해보려고 하세요. 처음엔 직접 하는 게 빠르더라도.
반복되면 생산성의 차이는 어마어마합니다.

"왜 하는가"가 중요하다

모호한 요청

"이 자료로 카드뉴스 하나 만들어줘"

목적 중심 요청

"신규 강좌 수강생 유치를 위한
전략적 홍보 콘텐츠를 기획하자"

주체성이 핵심

AI는 맘대로 하는 경향이 있습니다. 방향은 내가 잡고, 실행은 AI가 맡습니다.
What 전에 Why를 먼저 전달하세요.

파이프라인 사고

에이전트의 핵심은 업무를 하나의 흐름(Flow)으로 바라보는 것

INPUT

재료 / 데이터 / 맥락

→

PROCESS

AI Agent 수행

→

OUTPUT

결과물 / 가치

"단순히 일을 '하는' 것이 아니라, 일이 자동으로 흘러가는 길(Pipeline)을 뚫는 것.
그것이 에이전트의 진짜 가치입니다."

업무를 파이프라인으로 설계

한 번 만들면 재료만 바꿔 넣습니다

AI 친화적 환경 만들기

폴더 구조 · 마크다운 · Mermaid

AI가 잘 일하는 폴더 구조

AI에게 좋은 결과를 받으려면, 먼저 내 업무를 잘 정리해야 합니다

📁 my-project/ ├── input/ # 원본 자료 (재료) ├── output/ # AI가 만든 결과물 ├── reference/ # 참고 샘플·가이드 └── PRD.md # 목적과 요구사항

폴더 구조 3원칙

  • 의미 있는 폴더명 — 폴더명만으로 내용 유추 가능
  • 한 파일 = 한 주제 — AI 인용 최적화
  • 영어 + 대시(-) — 시스템 오류 방지

Tip: 폴더명은 한글보다 영문-대시 형태가 안전합니다. 예: card-news-agent

마크다운(Markdown) 기초

AI가 가장 잘 이해하는 텍스트 포맷 — 노션을 쓰고 계시다면 이미 쓰고 있는 겁니다!

# 제목 1 (가장 큰 제목) ## 제목 2 ### 제목 3 **굵게** *기울임* - 목록 항목 1 - 목록 항목 2 1. 번호 목록 1 2. 번호 목록 2 | 이름 | 역할 | |------|------| | 민지 | 마케터 |

왜 마크다운인가?

  • AI 기본 언어 — 모든 AI가 학습한 표준
  • 가볍고 빠름 — 어디서나 열림 (.md)
  • 범용 호환 — GitHub, Notion, VS Code
  • 버전 관리 — "최종_진짜최종" 탈출

VS Code에서 Mermaid 사용하기

확장 하나만 설치하면 마크다운 안에서 다이어그램을 바로 그릴 수 있습니다

설치 방법

  • VS Code에서 확장(Extensions) 열기
  • Markdown Preview Enhanced 검색 → 설치
  • Cmd/Ctrl + Shift + V로 미리보기
# .md 파일에 코드블록으로 작성 ```mermaid graph LR A[기획] --> B[초안] B --> C[검토] C --> D[배포] ``` # Cmd/Ctrl + Shift + V 로 미리보기!

Mermaid = 텍스트로 그리는 다이어그램. 코딩이 아니라 구조를 글로 적는 것입니다.

Mermaid 예시 — 업무 흐름도

# 플로우차트 ```mermaid graph TD A[강의 정보 입력] --> B[AI 분석] B --> C{채널 선택} C --> D[인스타그램] C --> E[블로그] C --> F[링크드인] D --> G[홍보 콘텐츠 완성] E --> G F --> G ```
# 마인드맵 ```mermaid mindmap root((내 업무)) 기획 리서치 브레인스토밍 기획서 작성 콘텐츠 제작 카피라이팅 디자인 의뢰 검수 배포 SNS 업로드 성과 분석 ```

실습 시간에 자기 업무 구조를 Mermaid로 직접 그려볼 겁니다!

3

"이 업무 흐름을 Mermaid 다이어그램으로 만들어줘"

실습 프롬프트 예시

# Step 1: 업무 파악 대화 "나는 [직업/역할]이야. 내 주요 업무를 파악해서 정리하고 싶어. 나한테 질문을 통해 업무 흐름을 파악해줘." # Step 2: Mermaid 시각화 "지금까지 파악한 내 업무 흐름을 Mermaid 플로우차트로 만들어줘. graph TD 형식으로."
# Step 3: D2 시각화 (선택) "같은 업무 흐름을 D2 다이어그램 문법으로도 만들어줘." # D2 문법 예시: 기획 -> 초안 작성 초안 작성 -> 검토 검토 -> 배포 # D2는 https://d2lang.com 에서 # 미리보기 가능!

완벽하지 않아도 됩니다! AI와 대화하면서 점점 구체화하세요.
"나랑 대화 많이 해줘" 하면 AI가 꼼꼼하게 질문합니다.
