AX 사고방식

AX (AI Transformation)란?

AX는 AI Transformation의 줄임말이에요. 단순히 AI를 써보는 것에 그치지 않고, 업무 과정 전체를 AI와 협업하는 중심으로 바꾸는 것을 뜻합니다.

화면을 보시면 세 단계가 있어요.

첫 번째, 기존 챗봇 사용 -- 가끔 질문할 때만 검색창처럼 사용하는 단계. 대부분 여기에 계세요.

두 번째, 전환점 -- AI의 가능성을 직접 실험하고 확인하는 단계.

세 번째, AX: 통합 협업 -- 모든 업무 프로세스에 AI를 녹여내는 단계.

핵심은 "AI에게 일을 시키는 것"이 아니라 "AI와 함께 일하는 것"이에요. 이 차이가 정말 큽니다.

10
AX 사고방식

AI를 동료로 대하기

AI를 진짜 잘 쓰는 사람들은 AI를 도구가 아니라 동료로 대합니다.

화면 왼쪽처럼 "카드뉴스 만들어줘" -- 이렇게 일방적으로 지시하면 뽑기 운이에요. 될 때도 있고 안 될 때도 있고.

오른쪽처럼 "이 데이터를 보고 같이 기획해보자" -- 이렇게 협업 관점으로 접근하면 일관된 품질의 결과물이 나옵니다.

뭐든 AI로 해보려고 하세요. 처음엔 직접 하는 게 빠르더라도요. 반복되면 생산성의 차이는 어마어마합니다.

11
AX 사고방식

"왜 하는가"가 중요하다

AI한테 일을 시킬 때 가장 중요한 건 "왜"예요.

"이 자료로 카드뉴스 하나 만들어줘" -- 이렇게 모호하게 요청하면 목적과 의도가 없으니 밋밋한 결과가 나와요.

반면에 "신규 강좌 수강생 유치를 위한 전략적 홍보 콘텐츠를 기획하자" -- 이렇게 목적을 분명히 하면 과정 전체의 밀도가 확 올라갑니다.

AI는 맘대로 하는 경향이 있어요. 방향은 내가 잡고, 실행은 AI가 맡는 겁니다. What 전에 Why를 먼저 전달하세요.

12
AX 사고방식

파이프라인 사고

에이전트의 핵심은 업무를 하나의 흐름(Flow)으로 바라보는 것이에요.

화면에 보이는 것처럼 딱 세 단계입니다.

INPUT -- 재료, 데이터, 맥락을 넣고

PROCESS -- AI 에이전트가 처리하고

OUTPUT -- 결과물, 가치가 나옵니다.

단순히 일을 "하는" 것이 아니라, 일이 자동으로 흘러가는 길(Pipeline)을 뚫는 것. 그것이 에이전트의 진짜 가치입니다.

13
AX 사고방식

업무를 파이프라인으로 설계

에이전트를 만든다는 건, 매번 도자기를 하나하나 손으로 빚는 게 아니라 자동화된 공장 컨베이어 벨트를 까는 것과 같아요.

한 번 라인을 잘 깔아두면, 나중엔 재료만 바꿔 넣으면 알아서 결과물이 나오는 거죠.

예를 들어 제가 만든 다채널 홍보 에이전트는 -- 강의 정보 MD 파일 하나만 넣으면, 채널별로 톤과 형식을 자동 조정해서, 인스타그램, 블로그, 링크드인 등 11개 채널의 홍보문구가 한 번에 나와요.

여러분 업무에도 이런 반복 패턴이 분명히 있을 거예요. 그걸 찾는 게 첫 걸음입니다.

14
AX 사고방식

AX 사고방식 정리

AX 사고방식을 세 가지로 정리하면요.

첫째, AI = 파트너. 도구가 아닌 협업 파트너로 대하세요.

둘째, Second Brain. 내 뇌의 한계를 AI로 확장하세요.

셋째, 파이프라인. INPUT에서 OUTPUT까지 자동으로 흘러가는 길을 만드세요.

자, 지금 자동화하고 싶은 반복 업무를 하나 떠올려 보세요. 뭘 넣을 건가(입력), 어떻게 바꿀 건가(처리), 뭘 받고 싶은가(출력). 이 세 가지가 에이전트의 출발점입니다.

AI가 잘 일하는 폴더 구조

AI에게 좋은 결과를 받으려면, 먼저 내 업무를 잘 정리해야 합니다.

폴더 구조는 딱 이렇게 잡으시면 돼요.

input 폴더 -- 원본 자료, 재료를 넣는 곳

output 폴더 -- AI가 만든 결과물이 담기는 곳

reference 폴더 -- 참고 샘플이나 가이드

그리고 PRD.md -- 목적과 요구사항을 정리한 파일

폴더 구조 3원칙도 기억해 주세요. 의미 있는 폴더명, 한 파일 = 한 주제, 그리고 영어 + 대시(-)로 이름 짓기. 한글 폴더명은 시스템 오류의 원인이 될 수 있어요!

참고: 예시 폴더명: card-news-agent, weekly-report-agent 등

AI 친화적 환경

마크다운(Markdown) 기초

마크다운은 AI가 가장 잘 이해하는 텍스트 포맷이에요.

혹시 노션(Notion) 쓰시는 분 계세요? 그러시다면 이미 마크다운을 쓰고 계신 겁니다!

문법이 정말 간단해요. # 하나면 큰 제목, ## 두 개면 중간 제목. **별표 두 개**로 감싸면 굵게. 이게 전부예요.

왜 마크다운이냐면 -- 모든 AI가 마크다운을 기본 언어로 학습했고, 파일이 가볍고, 어디서나 열리고, GitHub이나 Notion이나 VS Code 전부 호환돼요.

"최종_진짜최종.pptx" 파일 지옥에서도 벗어날 수 있습니다!

VS Code에서 Mermaid 사용하기

Mermaid는 텍스트로 다이어그램을 그리는 도구예요. 코딩이 아니라 구조를 글로 적는 것입니다.

VS Code에서 쓰려면 확장 하나만 설치하면 돼요. Markdown Preview Enhanced를 검색해서 설치하시고, Cmd+Shift+V로 미리보기하시면 됩니다.

화면 오른쪽에 코드를 보시면, 마크다운 파일 안에 코드블록으로 mermaid를 작성하면 끝이에요. graph LR, A에서 B로, B에서 C로... 이렇게 적으면 플로우차트가 그려집니다.

실습 시간에 여러분이 직접 이걸로 업무 구조를 시각화해볼 거예요!

AI 친화적 환경

Mermaid 예시 -- 업무 흐름도

Mermaid로 이런 것들을 그릴 수 있어요.

왼쪽은 플로우차트 예시예요. 강의 정보 입력에서 시작해서 AI 분석을 거쳐 채널별로 나뉘어 홍보 콘텐츠가 완성되는 흐름이죠.

오른쪽은 마인드맵 예시예요. "내 업무"를 중심으로 기획, 콘텐츠 제작, 배포로 나뉘고, 각각 세부 업무가 달려있죠.

이 코드를 직접 짤 필요는 없어요. Gemini에게 "내 업무 흐름을 Mermaid 다이어그램으로 만들어줘"라고 하면 알아서 만들어줍니다. 우리는 그걸 복사해서 붙여넣기만 하면 돼요!

데모 전에 코드편집기가 뭔지 한번 더 짚고 갈게요. 겁먹을 필요 전혀 없습니다!

코드편집기는 그저 우리 컴퓨터에 이미 있는 세 가지 기능을 한 화면에 모아둔 세트예요.

화면 왼쪽은 파일 탐색기 -- Finder나 윈도우 탐색기와 똑같아요.

가운데는 메모장 -- 텍스트 작성 도구.

아래쪽은 터미널 -- 명령어 입력 화면. 여기서 AI와 대화합니다.

VS Code는 Microsoft의 무료 편집기이고, Antigravity는 Google의 AI-First IDE예요. 둘 다 설치하셨죠? 오늘은 주로 VS Code에서 Gemini CLI를 써볼 거예요.

Demo: 다채널 홍보 에이전트 시연

(다채널 홍보 에이전트 프로젝트 폴더 열기)

이번에는 제가 실제로 업무에 사용하는 다채널 홍보 에이전트를 보여드릴게요.

이 에이전트는 정보 하나 넣으면 11개 채널 콘텐츠가 자동으로 만들어집니다.

INPUT -- 강의 정보가 담긴 MD 파일 하나

PROCESS -- 채널별로 톤과 형식을 자동 변환

OUTPUT -- 인스타그램, 블로그, 링크드인 등 11개 채널의 홍보문구

(에이전트 실행 시연)

아까 배운 파이프라인 사고가 여기서 그대로 적용되는 거죠. 한 번 파이프라인을 깔아놓으면, 재료만 바꿔 넣으면 됩니다.

참고: 한 건당 약 300원 비용 / 광고 없이 25만 조회수 달성 사례

라이브 데모

데모에서 본 것 정리

데모에서 확인하셨듯이, 에이전트가 주는 가치는 크게 세 가지예요.

속도 -- 10분이면 완성. 내가 실제로 손을 쓰는 시간은 1분도 안 돼요.

일관성 -- 매번 같은 품질. AI는 컨디션에 흔들리지 않습니다.

확장성 -- 하나 만들면 다른 업무에도 복제하고 응용할 수 있어요.

에이전트는 한 번에 안 돼요. 만들고, 테스트하고, 고치고. 이 반복이 전부예요. 근데 한 번만 해보면 "아, 되는구나" 싶은 순간이 옵니다.

실습: 나의 업무 구조 시각화

실습 순서는 세 단계예요.

Step 1 -- 터미널에서 gemini 입력해서 Gemini CLI를 실행하세요. VS Code든 Antigravity든 어디서든 OK!

Step 2 -- Gemini와 대화하면서 내 업무 구조를 정리해요. "나는 마케터인데, 내 주요 업무 흐름을 정리해줘. 질문을 통해 파악해줘" -- 이렇게 시작하시면 됩니다.

Step 3 -- 정리된 업무 흐름을 Mermaid 다이어그램으로 만들어달라고 요청하세요.

20분간 자유롭게 진행합니다. "완성"보다 "실행"에 초점! 일단 돌아가게 만드는 것이 목표입니다.

만든 Mermaid를 VS Code에서 보기

Gemini가 만들어준 Mermaid 코드를 VS Code에서 보는 방법이에요.
