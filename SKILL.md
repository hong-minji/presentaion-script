# HTML Presentation Generator

사용자의 발표 주제와 내용을 받아 **두 개의 연동된 HTML 파일**을 생성합니다:

1. **`{topic}-presentation.html`** — 청중에게 보여주는 슬라이드
2. **`{topic}-script.html`** — 발표자만 보는 컨트롤 화면 (스크립트, 타이머, 펜, 하이라이트 등)

두 파일을 같은 브라우저에서 각각 탭으로 열면 `BroadcastChannel`로 자동 동기화됩니다.

---

## 작업 순서

### 1단계: 콘텐츠 구조화

사용자에게 아래 정보를 확인합니다 (대화에서 이미 파악된 내용은 재질문하지 않습니다):

- 발표 제목
- 발표자 이름 / 소속
- 목표 발표 시간 (분)
- 대략적인 섹션 구성
- 발표 내용 (텍스트, 파일, URL 등)

### 2단계: 슬라이드 설계

콘텐츠를 섹션으로 나누고, 각 슬라이드의 타입과 컴포넌트를 결정합니다. 슬라이드 수는 발표 시간에 맞게 조절합니다 (대략 2-3분당 1슬라이드).

### 3단계: 두 HTML 파일 생성

`references/slide-components.md`를 읽고 슬라이드 HTML을 작성합니다.
`references/presenter-view-template.md`를 읽고 발표자 뷰 HTML을 작성합니다.

두 파일을 반드시 **같은 디렉토리**에 생성합니다.

### 4단계: 사용법 안내

파일 생성 후 사용자에게 안내합니다:

```
1. 프레젠테이션: {topic}-presentation.html을 브라우저에서 열기 (청중 화면)
2. 발표자 뷰: {topic}-script.html을 같은 브라우저의 새 탭에서 열기
3. 발표자 뷰에서 조작하면 프레젠테이션이 자동으로 따라갑니다
```

## 슬라이드 타입

| 타입 | 클래스 | 용도 |
|------|--------|------|
| 커버 | `.slide.cover` | 첫 슬라이드. 제목 + 부제 + 발표자 |
| 일반 | `.slide` | 제목 + 콘텐츠 조합 |
| 섹션 구분 | `.slide.slide--section` | 새 섹션 시작. 번호 + 제목 + 설명 |
| 데모 | `.slide.slide--demo` | 라이브 데모. 이모지 + 맥동 테두리 |
| 센터 | `.center-content` 내부 | Q&A, 핵심 인용문 등 |

## 콘텐츠 컴포넌트

| 컴포넌트 | 클래스 | 용도 |
|----------|--------|------|
| 카드 그리드 | `.grid.grid--2/3/4` + `.card` | 정보 카드 배열 |
| 하이라이트 박스 | `.highlight-box` | 핵심 메시지 강조 |
| 플로우 | `.flow` + `.flow-item` + `.flow-arrow` | 화살표로 연결된 흐름도 |
| 비교 | `.compare` + `.compare-item.good/.bad` | 좋은 예 vs 나쁜 예 |
| 코드 블록 | `.code-block` | 코드/터미널 표시 |
| 스텝 | `.steps` + `.step` | 번호가 있는 순서 단계 |
| 테이블 | `.table-wrap` + `table` | 데이터 테이블 |
| 리스트 | `.list` | 불릿 포인트 |
| 키커 뱃지 | `.kicker` | 작은 라벨 |
| 퍼즐 | `.puzzle-row` + `.puzzle-piece` | A + B = C 조합 |

각 컴포넌트의 정확한 HTML 마크업은 `references/slide-components.md`에 있습니다. 슬라이드를 작성하기 전에 반드시 읽어주세요.

## 발표자 뷰 기능

| 기능 | 설명 |
|------|------|
| 슬라이드 미리보기 | 왼쪽 75%에 iframe으로 16:9 프리뷰 |
| 발표 스크립트 | 오른쪽 25%에 슬라이드별 대본 |
| 타이머 | 시작/일시정지/리셋, 목표시간 대비 경과 표시 |
| 페이스 바 | 시간 진행률 vs 슬라이드 진행률 비교 |
| 펜 드로잉 | 슬라이드 위에 그리기 (청중 화면에 동기화) |
| 클릭 하이라이트 | 요소 클릭 시 스포트라이트 효과 (동기화) |
| 텔레프롬프터 | 대본을 큰 글씨로 표시 |
| 키보드 단축키 | 화살표(이동), T(텔레프롬프터), F(스크립트 전체화면) |

## 스크립트 작성 가이드

발표 스크립트는 실제로 말할 내용을 자연스러운 구어체로 작성합니다.

```html
<p>안녕하세요, 오늘 <span class="emphasis">핵심 키워드</span>에 대해 이야기해볼게요.</p>
<span class="action">(화면 전환: 데모 시작)</span>
<p>여기서 잠깐 <span class="pause">[2초]</span> 멈추고...</p>
```

| 서식 | 용도 |
|------|------|
| `<span class="emphasis">` | 핵심 키워드 강조 (파란색) |
| `<span class="action">` | 무대 지시 (회색 블록) |
| `<span class="pause">` | 쉬어가기 표시 |
| `<div class="script-ref">` | 참고 수치/출처 |

## 중요 규칙

1. **슬라이드 ID**: 모든 슬라이드에 `id="slide1"`, `id="slide2"`, ... 순서대로 부여
2. **첫 슬라이드에 `.active`**: `<div class="slide cover active" id="slide1">`
3. **JS 변수 일치**: `total = 슬라이드 수`, `sectionStarts = [각 섹션 시작 번호]`
4. **섹션 칩**: `data-slide` 속성은 해당 섹션의 첫 번째 슬라이드 번호
5. **파일 참조**: 발표자 뷰의 `PRESENTATION_FILE` 변수가 슬라이드 파일명을 참조
6. **같은 디렉토리**: 두 파일은 반드시 같은 폴더에 위치

## 테마 커스터마이징

CSS 변수로 전체 색상 테마를 변경할 수 있습니다:

```css
:root {
  --accent: #3b82f6;   /* 주요 강조색 */
  --bg: #0a0a0a;       /* 배경 */
  --surface: #141414;  /* 카드 배경 */
  --danger: #ef4444;   /* 경고/나쁜 예 */
  --success: #22c55e;  /* 성공/좋은 예 */
  --warning: #f59e0b;  /* 주의 */
}
```

사용자가 특정 색상이나 테마를 원하면 `--accent` 값을 변경합니다.
